# 워드 프로세싱 용어 및 기능

# 1. 워드 프로세서 기능

개념

- 워드 프로세싱: 문서를 작성하는 과정에서 작업할 수 있는 기술
- 워드 기능: 입력, 표시, 편집, 인쇄, 저장

워드 구성요소

- 입력 장치: 디지타이저(설계도면 입력), OCR(공공요금청구서),BCR(POS)
- 출력 장치: 플로터(설계도 출력)
- 저장 장치: 광 디스크

워드프로세서 특징

- 항시 보안에 유의하여야 한다(전송, 공유 가능)

키보드 기능

- 토글키(두개의 기능을 가진 키): insert, caps lock, num lock, scoll lock
- 조합키
- 기능키(F1-F12)
- 바로가기 키(단축키)

한글코드의 종류

- 완성형(정보 교환용)
- 조합형(정보 처리용)
- 유니코드(1005-1): 모든 문자를 2Byte, 국제 표준 코드

데이터 입력

- 한자음을 알 때: 음절, 단어, 문자 자동
- 한자음을 모를 때: 부수, 외자, 2스트로크

삽입/수정

- 삭제
    - back space: 누를 때마다 커서의 왼쪽방향으로 지움
    - delete: 누를 때마다 커서의 오른쪽 문자를 하나씩 삭제

영역지정

- 세번 클릭: 문단의 범위가 잡힘

❤ 문단: 글자입력을 시작해서 enter키 누를 때 까지를 말한다.

저장기능(주기억→보조)

확장자

- PDF: 컴퓨에 상관없이 공유, 활용되는 문서
- RTF: 서로 다른 프로그램에서 호환되는 서식 파일
- ARJ & ZIP: 압축 파일
- BAK: 백업 파일

워드 용어 정리

- 개체 연결 및 삽입(OLE): 다른 프로그램에서 작성한 그림/표를 연결하거나 
삽입하여 사용할 수 있게 하는 기능
- 사각형/원 중심은 Ctrl, 정사각형/정원은 Shift
- 상태 표시줄: 커서 위치, 쪽 번호, 삽입/수정 상태 등 여러 정보를 표시하는 줄
- 레이아웃: 문서의 내용을 페이지의 적당한 위치에 균형있게 배치하는 기능
- 행두금칙: 문장의 앞에 올 수 없는 문자()
- 매크로
- 각주: 각 페이지 하단에 표시하는 것
- 디폴트(기본값)
- 보일러 플레이트: 머리말, 꼬리말, 주석을 표시하기 위해 만든 구역
- 워드랩: 입력된 단어가 너무 길어 다음 줄로 이어질 경우 그 단어 전체를 다음 줄로 넘기는 것
- 래그드: 오른쪽 끝 열이 정렬되지 않은 상태
- 영문 균등: 단어 사이의 간격을 조절하여 문단의 양쪽 끝을 맞추는 기능
- 창 나누기: 하나의 화면을 여러 개로 분할
- 스풀: 인쇄할 때 다른 작업을 동시에 처리할 수 있는 기능(병행 처리)
- 하드 카피: 프린터에 출력
- 소프트 카피: 화면에 자료를 표시
- 문자 피치: 1인지에 포함되는 문자 수
- 치환
- 캡션: 표나 그림에 붙이는 제목
- 홈 베이스: 문서의 특정 위치로 바로 이동하는 기능

# 2. 문서 작성

정렬 기능

- SORT
    - 오름차순(가나다…123)
    - 내림차순(숫자→영문(소→대)→한글)

인쇄 기능

- 미리보기
- 문서의 일부분만 인쇄 가능
- 한 장에 여러 페이지 인쇄 가능(모아 찍기)
- 워터 마크 삽입 가능(저작권 표시)
- 파일로 인쇄(.PRN)

문자 크기(포인트)

- 전각(1:1), 반각(전각 문자의 1/2, 세로 길이는 같음)
- 양배(전각의 가로세로 2배, 총 4배), 첨자(전각의 1/4, ex: 지수)
- 장평: 글자의 가로 길이 조절, >100 ⇒ 납작, <100 ⇒ 길어짐

# 3. 문서 관리

문서의 분류

- 작성 주체에 따라: 공문서, 사문서
- 유통 대상에 따라: 대내문서, 대외문서, 전자문서
- 처리 단계에 따라: 접수문서, 배포문서, 기안문서

![image](https://github.com/youjiyeon/Word-Processor-Specialist/assets/57094856/8de72531-72f4-4f0a-a046-20d3dfa767da)

문서의 구성

사내문서(조직 내부 문서)

- 두문: 수신자명, 발신자명, 문서 번호, 발신 년 월 일(오른쪽 상단에 표시, 년월일 생략시 마침표로 구분)
- 본문: 제목, 주문(문서의 핵심, 간결하고 명확하게)
- 결문: 담당자명, 담당자 직위

사내 문서 종류

![image](https://github.com/youjiyeon/Word-Processor-Specialist/assets/57094856/f7127946-e28f-4034-8160-376103a6b2c8)

사외 문서(조직 외부의 수발되는 문서) 종류

- 거래 문서(주문서, 요청서)

교정 부호 표기

- 한번 교정된 부분도 다시 교정할 수 있다.

교정 부호 종류

![image](https://github.com/youjiyeon/Word-Processor-Specialist/assets/57094856/586a2648-5721-417a-bd77-1b5b6b9b68da)

한글 맞춤법

- 구개음화: ㄷ, ㅌ + -이, -히 ⇒ ㅈ, ㅊ으로 소리가 나더라도 ㄷ, ㅌ 표기
- 두음법칙: 녀, 뇨, 뉴, 니 → 요, 유, 이(양심, 역사, 예의)
- 파열음 표기에는 된소리를 쓰지 않음

문서의 논리

- 연역적 논리: 일반적인 원리 → 구체적 사실

문서 관리

- 문서 관리 목적: 문서 활용, 효율적인 공간 사용, 신속하게 문서를 찾음

문서 관리 원칙

문서 관리 표준화

문서 관리 절차

구분 → 분류 → 편철 → 보관 → 이관 → 보존 → 폐기

- 기산일: 문서 생산년도 다음해 1월 1일

문서 파일링

전자 문서 정의 

- 컴퓨터로 작성되어 전자적 형태로 송/수신 되는 문서
- 동시 열람 가능
- 공인 전자 문서 보관소(내용 및 송/수신 여부를 증명)

전자 결재 시스템

- 효율성 증대, 생산성 향상, 시간 단축, 효과적인 관리, 재가공 가능
- EDI: 컴퓨터 강에 표준화 된 포맷과 코드 체계를 이용하는 문서 교환 방식

전자 문서 관리 시스템(EDMS)

- 문서 작성부터 폐기까지 모든 과정을 통합적으로 관리

공문서의 구성 요소

- 두문: 행정 기관명, 수신자
- 본문: 제목, 내용, 붙임
- 결문

공문서의 항목 구분

1 → 가 → 1) →가) → (1) → (가) → ① → 동그라미 가

공문서 끝 표기

- 본문이 끝났을 경우: 한 자(2타) 띄고, ‘끝’ 표시
- 첨부물이 있는 경우: 붙임 하고 끝
- 본문 또는 첨부의 표시문이 오른쪽 한계선에서 끝났을 경우:
다음줄의 왼쪽 시작 위치에서 한 자(2타) 띄고, ‘끝’ 표시
- 표의 칸 중간에서 끝나는 경우: 표의 마지막 글자 다음 칸에 ‘이하 빈칸’, 끝 생략

공문서 수정

- 글자 중앙에 가로로 두선을 긋고 수정 후, 수정자가 서명 날인

공문서 내용 표기

![image](https://github.com/youjiyeon/Word-Processor-Specialist/assets/57094856/d4bb1145-090c-460c-b5fa-9c2b3c7a57cb)

공문서 종류

![image](https://github.com/youjiyeon/Word-Processor-Specialist/assets/57094856/40e98611-eefe-4b99-8313-4ee09fa7dd63)

공문서의 성립(결재권자의 결재가 있음으로서 성립)

- 서명의 종류: 전자문자, 전자이미지, 행정전자서명

문서의 효력 발생 시기(도달 주의)

- 일반 문서: 수신자에게 도달했을 때
- 전자 문서: 수신자의 컴퓨터에 팡이로 저장 됬을 때
- 공고 문서: 고시 또는 공고 후 5일이 경과한 때

공문서 기안

- 둘 이상의 행정기관장의 결재가 필요한 경우, 처리를 주관하는 기관에서 먼저 결재 후, 관련 행정기관의 결재를 받아 공동 명의로 시행한다.

업무 편람(안내서/지침서)

- 행정 편람, 직무 편람

결재

- 선람(선결): 결재가 순서대로
- 전결: 최고 책임자가 자기 권한의 일부를 일정한 자격자에게 위임
- 대결: 대신 결재, 나중에 결재 받아야 함(사후 보고)

문서 도장

- 관인
    - 청인: 행정기관의 명의
    - 직인: 행정기관장 or 보조기관
    - 민원 서류(등본, 초본)를 발급하는 경우 발급기관장 오른쪽 여백에 찍는다.

- 간인(2장 이상의 문서 앞장과 뒷장에 걸쳐 찍는 도장)
    - 허가, 인가, 등록

문서의 발신

- 정보통신망을 이용하여 발신하는 것이 원칙
